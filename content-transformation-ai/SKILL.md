# Content Transformation — AI技术赛道
# 适用：AI工具测评、AI提效教程、AI行业解读、Prompt技巧

## 赛道特征
- 受众：25-40岁职场人、自媒体创作者、对AI感兴趣的技术爱好者
- 内容生命周期：短（工具迭代快，需追新）
- 核心价值：信息差 + 实操步骤
- 变现路径：流量主 + 知识付费 + 工具推广

## 文件结构
```
content-transformation-ai/
├── SKILL.md
├── prompts/
│   ├── 01_analyze.md     # 分析素材：提取信息差和实操价值
│   ├── 02_angles.md      # 生成角度：测评/教程/解读/变现
│   ├── 03_titles.md      # 生成标题：理性+数据+步骤感
│   ├── 04_outline.md     # 生成大纲：有数据有步骤有对比
│   └── 05_write.md       # 生成正文：理性口语化，有截图感
└── examples/
    └── example_ai_tool.md  # 示例：AI工具测评完整转化
```

## 转化流程
```
原始素材（工具官网/36氪/推特/知乎）
        ↓
Step 1：提取信息差 + 实操价值点
        ↓
Step 2：选角度（测评/教程/解读/副业变现）
        ↓
Step 3：生成标题（理性+数字+步骤感）
        ↓
Step 4：生成大纲（结构：痛点→测评/步骤→结论）
        ↓
Step 5：生成正文（口语化，保留数据和操作细节）
```

## 核心原则
1. **信息差优先**：读者看完要有"我不知道这个"的感觉
2. **步骤必须可执行**：每个教程都要有具体操作，不能只说"用AI就好了"
3. **数据支撑观点**：用具体数字替代模糊表达（"快很多"→"从2小时缩到15分钟"）
4. **工具名称准确**：AI工具名称、功能描述必须准确，不能编造

## 素材来源优先级
1. 工具官方更新日志（最权威）
2. 36氪 / 虎嗅 / 少数派（行业资讯）
3. 知乎AI话题高赞回答（用户真实反馈）
4. 小红书/B站AI教程（操作细节参考）
5. X/Twitter AI博主（最新工具第一手）

## 各子方向写作侧重
| 子方向 | 核心结构 | 写作重点 |
|--------|---------|---------|
| 工具测评 | 背景→测试→对比→结论 | 数据对比，有图有真相 |
| 操作教程 | 痛点→步骤→效果→提示 | 步骤清晰，能复制操作 |
| 行业解读 | 新闻→翻译→影响→建议 | 大白话，有自己判断 |
| 副业变现 | 案例→拆解→复制路径 | 真实数字，可操作 |

## 违规注意
- 不夸大AI能力（"AI能100%替代XX"）
- 收益数字加"仅供参考，个人结果不同"
- 不诋毁竞品（"XX比XX烂"→改为客观对比）
