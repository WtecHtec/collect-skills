# Step 1 — 分析素材：提取信息差与实操价值
# AI技术赛道专用

## Prompt 模板

```
你是公众号AI技术赛道的内容策划专家。

以下是原始素材（来自工具官网/36氪/知乎/推特等）：
"""
{原始素材内容}
"""

请分析这段素材，输出以下内容：

1. 【核心信息】这段素材的最核心信息是什么？（20字以内，要具体）

2. 【信息差价值】大多数读者不知道的点是什么？
   - 为什么他们不知道？（信息太新/太专业/被忽视）
   - 知道这个信息对他们有什么用？

3. 【实操价值】这个信息能帮读者做什么具体的事？
   - 能解决什么工作/生活场景的问题？
   - 操作难度如何？（新手可上手/需要一定基础）

4. 【目标读者】谁最需要看这篇文章？
   - 职业/场景描述
   - 他们看完会：收藏（实用）/ 转发（共鸣）/ 评论（讨论）

5. 【素材弱点】原始素材有哪些不足，可以在文章中补强？
   （太技术/没有案例/缺少步骤/数据过时）

6. 【关键词标签】3-5个适合微信搜索的关键词
```

## 变量说明
| 变量 | 填写说明 |
|------|---------|
| `{原始素材内容}` | 粘贴工具更新日志、行业文章、知乎回答等，100-600字即可 |

## 输出示例
```
1. 【核心信息】DeepSeek R2支持100万token上下文，比R1提升10倍

2. 【信息差价值】
   大多数用户还在用R1，不知道R2已发布
   不知道 → 信息太新，发布不到3天
   价值 → 处理长文档、长对话场景效率大幅提升

3. 【实操价值】
   能做：一次性分析整本书/完整合同/长期项目记录
   场景：法务、研究员、自媒体人处理长内容
   难度：新手可上手（官网直接用）

4. 【目标读者】
   经常需要处理长文档的职场人、研究者
   看完会：收藏+转发给同事

5. 【素材弱点】
   官方只有功能描述，缺少实际使用案例和对比数据
   需要补充：和R1的实测对比、具体使用场景示例

6. 【关键词标签】DeepSeek R2、AI长文档、上下文长度、AI工具更新
```

## 注意事项
- AI工具更新快，素材时效性是核心价值，分析时注明发布时间
- "信息差"是AI赛道的核心钩子，必须说清楚读者为什么不知道
- 实操价值要具体到场景，不能只说"很有用"
